# Implementation Plan: Twitter (X) Integration

**Branch**: `005-twitter-x-integration` | **Date**: 2026-02-21 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-twitter-x-integration/spec.md`

---

## Summary

Extend the Personal AI Employee with Twitter/X monitoring and auto-posting capabilities following the established Playwright session-based pattern (Feature 004: Meta Social Integration). The implementation adds a `TwitterWatcher` (perception layer) and `TwitterPoster` (action layer), integrates them into the orchestrator and action executor, extends the content scheduler with Twitter-specific short-form templates (≤280 chars), and creates a `skills/twitter-manager/SKILL.md` skill definition.

**Architecture pattern**: Mirrors Feature 004 exactly — persistent Playwright context at `config/twitter_session/`, `BaseWatcher` inheritance, `_scan_notifications()` + `_scan_messages()` + `create_action_file()`, poster `process_approved_posts()` with 280-char validation.

---

## Technical Context

**Language/Version**: Python 3.13+
**Primary Dependencies**: playwright (async_api), python-dotenv, pyyaml (via existing utils), pytest, pytest-asyncio, ruff (all already installed)
**Storage**: Vault markdown files (`.md`), JSON dedup store (`vault/Logs/processed_twitter.json`), Playwright persistent browser profile (`config/twitter_session/`)
**Testing**: pytest + pytest-asyncio (existing test suite at `tests/`)
**Target Platform**: Windows 11 (local), cross-platform compatible
**Project Type**: Single Python project (existing structure)
**Performance Goals**: Watcher cycle ≤ 30s for DEV_MODE; browser-based cycle ≤ 60s for 20 notifications + 15 DMs
**Constraints**: 280-char hard limit enforced in both template authoring and poster validation; no silent truncation; session-based only (no API tokens)
**Scale/Scope**: Single user, local execution; up to 5 tweets/day (constitution rate limit)

### TwitterWatcher Class

```python
class TwitterWatcher(BaseWatcher):
    """Watches Twitter/X notifications and DMs, creates vault action files."""

    def __init__(
        self,
        vault_path: str,
        session_path: str = "config/twitter_session",
        check_interval: int = 300,
        keywords: list[str] | None = None,
        headless: bool = False,
        dry_run: bool = True,
        dev_mode: bool = True,
    ) -> None: ...

    # Browser Management (identical to FacebookWatcher)
    async def _launch_browser(self) -> None: ...
    async def _close_browser(self) -> None: ...
    async def _ensure_browser(self) -> None: ...
    async def _navigate_and_wait(self, url: str, wait_seconds: float = 8.0) -> None: ...

    # Session State (Twitter-specific selectors)
    async def _check_session_state(self) -> str: ...
    # Returns: 'ready' | 'login_required' | 'captcha' | 'unknown'
    # Login check: '/i/flow/login' or '/login' in URL
    # Auth check: [data-testid="AppTabBar_Home_Link"] present
    # Fallback:   [data-testid="primaryColumn"] present

    async def setup_session(self) -> bool: ...
    # Navigate to https://x.com/home, prompt user to login, wait for Enter

    # Deduplication (identical to FacebookWatcher)
    def _load_processed_ids(self) -> None: ...
    def _save_processed_ids(self) -> None: ...
    def _cleanup_old_ids(self) -> None: ...

    # Scanning
    async def _scan_notifications(self) -> list[dict[str, Any]]: ...
    # URL: https://x.com/notifications
    # Primary selector: article[data-testid="notification"]
    # Fallback: [data-testid="notificationItem"], article[data-testid="tweet"]

    async def _scan_messages(self) -> list[dict[str, Any]]: ...
    # URL: https://x.com/messages
    # Primary selector: [data-testid="conversationItem"]
    # Fallback: [data-testid="DMConversationItem"], [data-testid="conversation"]

    # Core Loop
    async def check_for_updates(self) -> list[dict[str, Any]]: ...
    async def create_action_file(self, item: dict[str, Any]) -> Path | None: ...
    async def run(self) -> None: ...
```

### TwitterPoster Class

```python
class TwitterPoster:
    """Publishes approved Twitter posts from the vault."""

    TWITTER_CHAR_LIMIT = 280

    # Selectors (data-testid based — research-verified)
    POST_SELECTORS = {
        "text_area": [
            "div[data-testid='tweetTextarea_0']",
            "div[role='textbox'][aria-label*='tweet' i]",
            "div[aria-label='Post text']",
            "div[contenteditable='true']",
        ],
        "submit_button": [
            "button[data-testid='tweetButtonInline']",
            "button[data-testid='tweetButton']",
            "div[data-testid='tweetButtonInline']",
            "button[aria-label='Post']",
        ],
    }

    AUTHENTICATED_SELECTORS = [
        "[data-testid='AppTabBar_Home_Link']",
        "[data-testid='primaryColumn']",
        "[data-testid='sidebarColumn']",
        "a[href='/compose/post']",
    ]

    def _validate_post(self, body: str, fm: dict) -> str | None:
        # Returns None if valid, rejection_reason string if invalid
        if not body.strip(): return "empty_body"
        if len(body) > 280: return "exceeds_character_limit"

    async def _publish_post(self, body: str) -> bool:
        # 1. Navigate to https://x.com/home
        # 2. Check session state → fail fast if not ready
        # 3. Find & click textarea (auto-focused on home page)
        # 4. Type body text
        # 5. Click Post button (tweetButtonInline)
        # 6. Wait 3s for confirmation

# In ActionExecutor.HANDLERS:
HANDLERS: dict[str, str] = {

    "instagram_post": "_handle_instagram_post",
    "twitter_post": "_handle_twitter_post",   # NEW

async def _handle_twitter_post(
    self, file_path: Path, _fm: dict[str, Any], _cid: str
) -> None:
    from backend.actions.twitter_poster import TwitterPoster
    poster = TwitterPoster(
        vault_path=str(self.vault_path),
        session_path=os.getenv("TWITTER_SESSION_PATH", "config/twitter_session"),
        headless=os.getenv("TWITTER_HEADLESS", "false").lower() == "true",
        dry_run=self.config.dry_run,
        dev_mode=self.config.dev_mode,
    )
    try:
        count = await poster.process_approved_posts()
    finally:
        await poster._close_browser()
    if count == 0:
        raise RuntimeError(f"TwitterPoster processed 0 posts for {file_path.name}")
```

def draft_exists_today(self, today: str | None = None) -> bool:
    today = today or self.today_str()
    platforms = ("LINKEDIN", "FACEBOOK", "INSTAGRAM", "TWITTER")  # Add TWITTER
    for subdir in ("Pending_Approval", "Approved"):
        for prefix in platforms:

    return False

## Key Constraints

1. **280 chars is hard** — never silent truncation in poster; poster always rejects if exceeded
2. **Session is separate** — `config/twitter_session/` is Twitter-only (not shared with Meta)
3. **Template-based only** — no LLM-generated content; all tweet content from pre-authored templates
4. **HITL required** — `type: twitter_post` files MUST be in `vault/Approved/` to be processed
5. **DEV_MODE default** — `DEV_MODE=true` must be default; requires explicit `DEV_MODE=false` for real posting
