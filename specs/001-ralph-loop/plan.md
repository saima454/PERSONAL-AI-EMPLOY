## Summary

Implement the Ralph Wiggum stop-hook pattern — a loop controller that keeps Claude Code iterating on multi-step tasks until they complete or a safety limit is reached. Two execution modes: (1) **Hook-based** — `onStop` hook in `.claude/settings.json` intercepts Claude Code's interactive exit and blocks it if the task is incomplete; (2) **Subprocess-based** — `ralph_loop.py` invokes `claude -p` programmatically and manages loop state in Python. Both modes share `state_manager.py` (vault state persistence) and `prompt_injector.py` (context continuity). Two completion strategies: promise marker in output or file presence in `vault/Done/`. Safety: max iterations, per-iteration timeout, total timeout, emergency stop via `vault/STOP_RALPH`.

## Technology Decisions

### Decision A: Two-Mode Architecture

**Decision**: Implement both hook-based and subprocess-based modes.

**Hook mode** (interactive): `.claude/settings.json` `onStop` hook → `stop_hook.py` reads state file → blocks or allows exit.

**Subprocess mode** (programmatic): `ralph_loop.py` calls `claude -p "prompt" --output-format json --resume SESSION_ID`. Uses `asyncio.to_thread` for cross-platform compatibility.

**Rationale**: Spec requires both orchestrator auto-spawn (needs subprocess mode) and stop hook interception (needs hook mode). Shared `state_manager.py` and `prompt_injector.py` eliminate code duplication.

### Decision B: State Persistence After Every Iteration

**Decision**: `state_manager.py` writes state file after every iteration (not just at terminal states). Ensures recovery from crashes and enables `--status` to show in-progress loops.

**Rationale**: Constitution Principle VIII requires system to be "recoverable from any error state". Writing state after each iteration means worst case = 1 iteration of lost work.

---

### Decision C: `asyncio.wait_for` for Timeout

**Decision**: Wrap `claude -p` subprocess in `asyncio.wait_for(coro, timeout=per_iteration_timeout)`. Background `asyncio.Task` monitors `vault/STOP_RALPH` sentinel every 1 second.

**Rationale**: Matches `orchestrator.py` pattern exactly. `contextlib.suppress(asyncio.CancelledError)` for clean shutdown. Cross-platform (no `signal.alarm`).

---

### Decision D: `--resume SESSION_ID` for Context Continuity

**Decision**: Capture `session_id` from first `claude -p --output-format json` call. Pass `--resume SESSION_ID` to all subsequent iterations. Additionally, `prompt_injector.py` prepends a summary of previous iteration outputs to the prompt.

**Rationale**: `--resume` preserves Claude's tool call history and conversation context. Prompt injection provides explicit summary for iterations where session may be lost (DEV_MODE, network interruption).

---

### Decision E: DEV_MODE Simulation

**Decision**: In DEV_MODE, `ralph_loop.py` simulates each iteration as: `await asyncio.sleep(1)` + generate fake output. Iteration 3 (or final) outputs the configured completion marker to demonstrate clean termination.

**Rationale**: Same pattern as `briefing_generator.py` DEV_MODE. Loop state machine fully exercised without calling `claude -p`.

### `backend/ralph_wiggum/stop_hook.py`

```python
def main() -> None:
    """Read stdin JSON, check task state, output block/approve JSON."""
    payload = json.loads(sys.stdin.read())
    claude_project_dir = Path(payload.get("claude_project_dir", "."))
    vault_path = claude_project_dir / "vault"

    mgr = StateManager(vault_path)

    # Emergency stop takes priority
    if mgr.emergency_stop_active():
        print(json.dumps({"decision": "block", "reason": "Emergency stop active (vault/STOP_RALPH exists). Remove file to resume."}))
        return

    # Find most recent in_progress task
    tasks = mgr.load_all_tasks()
    active = [t for t in tasks if t.status == LoopStatus.in_progress]

    if not active:
        print(json.dumps({"decision": "approve"}))
        return

    task = active[-1]  # most recent

    # Check safety limits
    if task.current_iteration >= task.max_iterations:
        task.status = LoopStatus.halted
        task.halt_reason = HaltReason.max_iterations_reached
        mgr.update_task(task)
        print(json.dumps({"decision": "approve"}))  # allow exit; task is halted
        return

    # Task incomplete — block exit
    task.current_iteration += 1
    mgr.update_task(task)
    reason = (
        f"Task {task.task_id} incomplete. "
        f"Iteration {task.current_iteration}/{task.max_iterations}. "
        f"{PromptInjector.build_continuation_prompt(task.prompt, task.iterations)}"
    )
    print(json.dumps({"decision": "block", "reason": reason}))

if __name__ == "__main__":
    main()
```
